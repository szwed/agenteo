# Getting Started

Entry points by stack level:

## For Baremetal

- [Baremetal Automation](https://ironcore.dev/baremetal/) -- manage physical servers K8s-natively
- [IronCore-in-a-Box](https://github.com/ironcore-dev/ironcore-in-a-box) -- local kind cluster demo

## For Existing Infrastructure

- [Gardener](https://gardener.cloud/docs/gardener/deployment/setup_gardener/) -- managed K8s across [multiple infra providers](https://gardener.cloud/#everywhere-you-want-it); also [local setup](https://gardener.cloud/docs/gardener/deployment/getting_started_locally/)
- [Garden Linux](https://github.com/gardenlinux/gardenlinux) -- minimal, auditable K8s node OS

## For Existing Kubernetes Clusters

- **Platform Mesh** via [kcp](https://docs.kcp.io/kcp/main/setup/), [OpenMFP](https://openmfp.org/documentation/getting-started/), [OpenFGA](https://openfga.dev/docs/getting-started/setup-openfga/overview)
- [Open Component Model (OCM)](https://ocm.software/docs/getting-started/) -- software delivery across environments
- [OpenBao](https://openbao.org/docs/install/) -- secrets and PKI management

## For Existing Workloads

- [Open Resource Discovery](https://open-resource-discovery.github.io/specification/introduction) -- publish app/service metadata
- [OCM](https://ocm.software/docs/getting-started/) -- software delivery standard and tooling

## Next Steps

- **Assemble** -- see [[Showroom-Scenarios]] for working integration examples
- **Adapt** -- most components are extensible for your infrastructure
- **Pick and choose** -- use only what provides value for your use case
